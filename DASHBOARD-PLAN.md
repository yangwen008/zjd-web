## 一、前台栏目 → 发布权限 → 后台对应关系

### 1.1 前台 8 大栏目与发布者角色

| 前台栏目 | 数据表 | source_type | 谁能发布 | 发布入口 | 审核流程 |
|---------|--------|-------------|---------|---------|---------|
| ① 首页数据概览 & 省级流速 | `market_data` | — | **仅 admin** | `/admin/market-data` | 无需审核 |
| ② 核心地点寻源区 | `assets` | 任何 | **任何角色** | 按角色不同表单 | 需审核 |
| ③ 纯净一手官方原矿区 | `assets` | `official` | **仅 admin** | `/admin/assets` 新增 | 无需审核（admin直接上架） |
| ④ 村集体直发专区 | `assets` | `village` | **仅 village** | `/dashboard/assets/new` | 需审核（管理员验证村委授权书） |
| ⑤ 文旅大宗产业路演带 | `bulk_projects` | — | **仅 admin**（指定用户） | `/admin/bulk-projects` | 无需审核 |
| ⑥ 数字化隐居基建硬指标 | `infrastructure_ratings` | — | **仅 admin**（指定用户） | `/admin/infra-ratings` 或 `/dashboard/infra` | 无需审核 |
| ⑦ 本地金牌农房合伙人 | `brokers` | — | **仅 admin** | `/admin/brokers` | 无需审核 |
| ⑧ 解锁终端 | `unlock_tasks` | — | 买家触发 | 前台自动 | — |

### 1.2 发布权限矩阵（核心规则）

```
谁能发布什么 → 前台哪个栏目显示 → 需要什么字段 → 什么审核流程
```

| 角色 | 能发布到 | 前台栏目 | 表单差异 | 审核 |
|------|---------|---------|---------|------|
| **admin** | assets (official) | 官方原矿区 | 完整字段，直接上架 | 无需 |
| **admin** | assets (village) | 村委直发区 | 完整字段 + 村委授权书 | 无需 |
| **admin** | bulk_projects | 大宗路演 | 大宗专属字段 | 无需 |
| **admin** | market_data | 行情数据 | 行情专属字段 | 无需 |
| **admin** | infrastructure_ratings | 基建指标 | 基建专属字段 | 无需 |
| **admin** | brokers | 合伙人 | 合伙人专属字段 | 无需 |
| **village** | assets (village) | 村委直发区 | 简化字段 + **必须上传村委授权书** | **需审核** |
| **broker** | assets (ugc) | 合伙人管辖资产 | 简化字段 + 合伙人关联 | **需审核** |
| **buyer** | assets (ugc) | 搜索结果 | 最简字段 | **需审核** |
| **infra_editor** | infrastructure_ratings | 基建指标 | 基建专属字段 | **需审核** |

### 1.3 关键业务规则

```
规则1：官方原矿 (source_type=official)
  → 只有 admin 能发布
  → 直接 status=approved，前台立即展示
  → 来源标注"一手官方资源"

规则2：村委直发 (source_type=village)
  → village 角色发布 → status=pending → 管理员审核
  → admin 发布 → status=approved（直接上架）
  → 必须上传"村委授权书"（R2 /private/certificates/）
  → 审核时管理员核验授权书真实性
  → 来源标注"村委官方直招"

规则3：合伙人 UGC (source_type=ugc)
  → broker 角色发布 → status=pending → 管理员审核
  → 发布时自动关联 broker_id
  → 前台在合伙人详情页"管辖资产"中展示
  → 来源标注"经纪人发布"

规则4：大宗路演 (bulk_projects 表)
  → 只有 admin 能发布（或 admin 指定的"大宗发布者"角色）
  → 使用独立的 bulk_projects 表，不用 assets 表
  → 前台在"文旅大宗产业路演带"展示

规则5：基建评分 (infrastructure_ratings 表)
  → admin 或 infra_editor 角色可录入
  → infra_editor 发布需审核
  → 前台在"基建硬指标排行榜"展示

规则6：行情数据 (market_data 表)
  → 仅 admin 可操作
  → 前台在"流转大盘"展示

规则7：合伙人 (brokers 表)
  → 仅 admin 可管理
  → 前台在"金牌合伙人"展示

规则8：普通用户发布
  → buyer 角色发布 → source_type=ugc → status=pending → 管理员审核
  → 最简表单（标题、地址、面积、价格、联系方式）
  → 通过搜索结果展示
```

### 2.3 Session 管理

```typescript
// 登录流程
POST /api/auth/login { phone, password }
  → 查 users 表
  → 验证密码（Web Crypto SHA-256）
  → 生成 session token (UUID)
  → 写入 user_sessions 表
  → 设置 cookie: user_session=<token>; HttpOnly; Secure; SameSite=Strict

// 注册流程
POST /api/auth/register { phone, password, nickname, role_apply }
  → 检查手机号不重复
  → 密码加密
  → INSERT users (phone, password_hash, nickname, role='buyer', status='active')
  → 如申请 broker/village → 额外记录 role_apply，status 不变，等管理员审核
  → 自动登录

// 退出流程
POST /api/auth/logout
  → 删除 session
  → 清除 cookie

// 获取当前用户
GET /api/auth/me
  → 从 cookie 读取 session
  → 查 user_sessions + users
  → 返回用户信息 + permissions
```

## 四、API 设计

### 4.1 认证 API

| 路由 | 方法 | 说明 |
|------|------|------|
| `/api/auth/register` | POST | 注册 { phone, password, nickname, role_apply? } |
| `/api/auth/login` | POST | 登录 { phone, password } → set cookie |
| `/api/auth/logout` | POST | 退出 → clear cookie |
| `/api/auth/me` | GET | 获取当前用户信息 |

### 4.2 Dashboard API（数据隔离）

| 路由 | 方法 | 说明 | 数据隔离 |
|------|------|------|---------|
| `/api/dashboard/stats` | GET | 我的统计 | `WHERE user_id = ?` |
| `/api/dashboard/assets` | GET | 我的资产列表 | `WHERE user_id = ?` |
| `/api/dashboard/assets` | POST | 发布新资产 | 写入 `user_id = 当前用户` |
| `/api/dashboard/assets/[id]` | PUT | 编辑我的资产 | `WHERE user_id = ?` |
| `/api/dashboard/assets/[id]` | DELETE | 删除我的资产 | `WHERE user_id = ?` |
| `/api/dashboard/bulk-projects` | GET | 我的大宗项目 | `WHERE user_id = ?` |
| `/api/dashboard/bulk-projects` | POST | 发布大宗项目 | 写入 `user_id = 当前用户` |
| `/api/dashboard/leads` | GET | 我的线索 | 通过 `assets.user_id = ?` 关联 |
| `/api/dashboard/favorites` | GET | 我的收藏 | `WHERE user_id = ?` |
| `/api/dashboard/favorites` | POST | 添加/取消收藏 | `WHERE user_id = ?` |
| `/api/dashboard/profile` | GET | 个人资料 | `WHERE id = ?` |
| `/api/dashboard/profile` | PUT | 修改资料 | `WHERE id = ?` |
| `/api/dashboard/infra` | GET | 我负责的基建数据 | `WHERE province IN (我的负责区域)` |
| `/api/dashboard/infra` | POST | 录入基建数据 | 写入 + 标记待审核 |
